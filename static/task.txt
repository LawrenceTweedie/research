Заглавная страница

При нажатии на длинную строку «Выберите рынок…. и тд» должен открываться раскрывающийся список со всеми названиями рынков с возможностью динамического поиска, чтоб по введенным буквам искались подходящие по названию рынки.
После выбора рынка открывается возможность выбрать регион с динамическим списком в строке справа или оставить «Вся Россия». !!!! Без выбора рынка выбрать регион нельзя !!!!!.

Страницы рынков
Как уже говорилось, каждый рынок в каждом регионе – отдельная страница.
Название рынка и региона все также берется из файла search.json. Ссылка формируется следующим образом: inresearch.ru/id рынка/id региона. Ссылка без id региона означает, что анализ рынка по всей России (как пример ниже). Также у пользователя есть возможность выбрать другой регион в раскрывающемся списке.

Дальше идет блок с видами деятельности

Здесь стоит рассказать о структуре файлов на сервере. Домен inresearch.ru зарегистрирован на меня в Beget, там же облачное хранилище. Хранилище состоит из папок с id рынка, внутри папки с id рынка вся необходимая по рынку информация. Виды деятельности лежат в папке «id рынка_okv.json»

Этот раздел очень важен для SEO, так как люди обычно гуглят «Рынок *название вида деятельности* в *название региона*», поэтому на сайт нужно затолкать все виды деятельности, указанные в файле.
Дальше блок с анализом новостей с помощью ИИ.

В каждой папке с рынком также лежит файл id рынка_news.json. Скрипт автоматически собирает новости, сохраняет заголовки и ссылки на новости, AI их автоматически оценивает и дает emotion. Файлы автоматически обновляются раз в месяц, на сайте нужно настроить автоматическое обновление заголовков, привязанных к ним ссылок и emotion, который дает AI. Важно отметить, что здесь нет разделения на регионы. То есть информация должна быть одна и та же у всех регионов одного рынка.

В каждой папке рынка лежит файл «id рынка_top10.json». У него следующая структура

Данный файл используется только если рынок анализируется по всей России. Также в папке лежит файл «id рынка_regions_top10.json»

Здесь представлены топ-10 компаний для каждого региона в анализируемом рынке. То есть если рынок «Сбор грибов» анализируется в регионе «Москва», мы заходим в папку с названием «id рынка», внутри находим файл «id рынка_regions_top10.json», внутри находим ключ «Москва» и оттуда вытаскиваем топ-10 компаний и вставляем на сайт.
Если компаний меньше 10, то заполняем нужное количество, остальное – прочерки во всех значениях

Вид гистограммы и визуальное оформление можно взять другим, если та форма, что сейчас в дизайне не реализуема.
Все показатели рынка хранятся в файле «id рынка_metrics.json»:

Все показатели, кроме одного, представлены за 5 лет (с 2020 по 2024). На сайт нужно вставить название показателя, годы и значения. А также построить интерактивный график-гистограмму, чтоб при наведении на шкалы подсвечивалось год и значения. Значения показателей меняться не будут, но могут дополняться новые показатели, поэтому надо сделать так, чтоб в случае появлении новых показателей, их можно было добавить на сайт.
Для регионов данные лежат в файле «id рынка_region.json»

То есть если рынок «Добыча угля» анализируется в России, то нужно зайти в папку с его id, найти файл «id рынка_region.json», найти нужный регион и вставить оттуда на сайт. В регионах данные также за 5 лет.
